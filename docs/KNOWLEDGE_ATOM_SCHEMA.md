# Knowledge Atom Schema

## Purpose

A Knowledge Atom is the smallest reviewable TFUGAG research unit. It converts raw inspiration into a bounded object that can be classified, tested, compared, archived, or promoted.

## Required fields

```yaml
title: short descriptive title
claim: one precise statement
status: S | E | X | D | C | A
assumptions:
  - required condition or premise
evidence:
  - proof, calculation, test, citation placeholder, or observation
limitations:
  - boundary, uncertainty, counterexample, or risk
next_action: smallest useful next step
```

## Status meanings

- `S` speculative: intuition only.
- `E` exploratory: clarified, but weakly formalized.
- `X` crystallizable: has definitions, assumptions, and a validation path.
- `D` demonstrated: has local proof, simulation, benchmark, or reproducible calculation.
- `C` canonical: stable, reused, low-duplication part of the repository.
- `A` archived: superseded, historical, or intentionally inactive.

## Promotion requirements

| From | To | Requirement |
| --- | --- | --- |
| S | E | Clarify the intuition into one claim. |
| E | X | Add assumptions, limits, and validation path. |
| X | D | Add proof, test, simulation, benchmark, or calculation. |
| D | C | Show stable reuse and low duplication. |
| Any | A | Explain why the object is archived. |

## OAK review checklist

Before promotion, answer:

1. Is the claim one statement rather than a cluster of ideas?
2. Are assumptions explicit?
3. Are limitations honest?
4. Is evidence appropriate to the status?
5. Is the next action small enough to execute?
6. Is there any overclaiming language that should be reduced?

## Markdown template

```markdown
# <Knowledge Atom Title>

Status: `E`

## Claim

<One precise claim.>

## Assumptions

- <Assumption 1>

## Evidence

- <Evidence or placeholder.>

## Limitations

- <Boundary or uncertainty.>

## Next action

<Smallest useful step.>

## OAK risk flags

- <Generated by review kernel or manually listed.>
```

## Example

```yaml
title: OAK Air-Gap governance kernel
claim: AI-generated TFUGAG artifacts can be classified before promotion into public canon.
status: X
assumptions:
  - Artifacts can be represented as bounded Knowledge Atoms.
  - Human review remains required before canonization.
evidence:
  - Minimal Python classifier exists.
  - Unit tests cover safe and unsafe cases.
limitations:
  - This is governance logic, not scientific proof validation.
next_action: Add reviewed examples from repository material.
```
