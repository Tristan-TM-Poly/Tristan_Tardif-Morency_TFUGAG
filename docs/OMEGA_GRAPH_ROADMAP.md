# TFUGAG Omega Graph Roadmap

## Vision

Unify all subsystems around a single OmegaGraph schema.

Every subsystem becomes a projection of the same graph:

- Evidence Layer
- Knowledge Layer
- Discovery Layer
- Governance Layer
- Evolution Layer
- Atlas Layer

## Phase 1

Map existing entities to OmegaNode kinds.

| Existing Concept | OmegaNode Kind |
|------------------|----------------|
| Knowledge Atom | atom |
| Claim | claim |
| Evidence | evidence |
| Question | question |
| Protocol | protocol |
| Result | result |
| Bridge | bridge |

## Phase 2

Create adapters:

- evidence_to_omega
- claims_to_omega
- atlas_to_omega
- protocols_to_omega

## Phase 3

Frontier Analysis

Compute:

- evidence density
- contradiction density
- bridge density
- novelty density

Generate frontier nodes.

## Phase 4

Evolution Layer

Represent node history as temporal trajectories.

node(t0)
node(t1)
node(t2)

## Long-Term Goal

Transform TFUGAG into a unified epistemic operating system built on OmegaGraph.
