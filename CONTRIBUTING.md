# Contributing

This repository follows the DCT-Omega standard for TFUGA ecosystem work.

## Contribution rule

Every contribution should improve at least one DCT-Omega axis:

- Document: definitions, assumptions, explanations, limits;
- Code/Calcul: executable kernel, symbolic derivation, notebook, or reproducible calculation;
- Test: proof, simulation, validation, audit, or reproducibility trace;
- Risk: epistemic debt, uncertainty, misuse, duplication, or safety limits;
- Status: S, E, X, D, C, or A;
- Next action: smallest concrete continuation.

## Status labels

- S: speculative;
- E: exploratory;
- X: crystallizable;
- D: demonstrated locally;
- C: canonical;
- A: archived.

## Pull request checklist

- [ ] I stated the status of the contribution.
- [ ] I separated stable canon, crystallizable branch, exploratory branch, and archive material.
- [ ] I documented assumptions, limits, and next action.
- [ ] I added tests, examples, proofs, or audit traces when relevant.
- [ ] I avoided promoting unverified claims to canonical status.
